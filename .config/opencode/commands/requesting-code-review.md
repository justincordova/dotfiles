# /requesting-code-review

When completing tasks, implementing major features, or before merging. Verifies work meets requirements and catches issues.

## When to Use
- Completing tasks or major features
- Before merging to main
- Need peer review for quality assurance
- Want to verify work meets requirements

## Quick Overview

**Core Principle:** Get feedback before merging to ensure quality.

**Process:**
1. Prepare work for review (clean, documented, tested)
2. Create pull request or request review
3. Address feedback
4. Verify integration

## Key Principles
- Code should be review-ready (clean, tested, documented)
- Get review before merging, not after
- Address feedback thoughtfully
- Verify changes work before finalizing
