# /finishing-a-development-branch

When implementation is complete, all tests pass, and you need to decide how to integrate the work. Presents structured options for merge, PR, or cleanup.

## When to Use
- Implementation is complete and tested
- Need to integrate work into main branch
- Deciding between merge strategies

## Quick Overview

**Core Principle:** Present integration options and complete work systematically.

**Process:**
1. Verify implementation complete
2. Present integration options (merge, PR, cleanup)
3. Execute chosen strategy
4. Verify integration

## Key Principles
- Don't complete until all tests pass
- Present options, don't assume preference
- Follow git conventions from the project
- Clean up if needed
