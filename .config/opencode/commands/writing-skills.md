# /writing-skills

When creating new skills, editing existing skills, or verifying skills work before deployment. Provides domain-specific instructions and workflows.

## When to Use
- Creating new skills
- Editing or improving existing skills
- Verifying skills work correctly
- Learning how to write skills

## Quick Overview

**Core Principle:** Skills are code - write, test, verify, deploy.

**Process:**
1. Understand skill purpose and scope
2. Write skill content following format
3. Test skill behavior
4. Verify skill works as expected
5. Deploy skill

## Skill Format
- Frontmatter with name and description
- Clear overview
- When to use (scenarios)
- Process/workflow
- Key principles
- Examples
- Anti-patterns to avoid
- Related skills

## Key Principles
- Skills should be domain-specific
- Provide clear workflows
- Include examples and anti-patterns
- Test skills before deployment
- Keep skills focused and actionable

## Anti-Pattern
- Vague or generic skills
- No clear when-to-use guidance
- Missing examples
- Untested skills
- Overly broad scope

- Review existing skills as examples
